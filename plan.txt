🚀 Пошаговый план разработки Lantern Messenger
🟢 Фаза 1: Каркас, Локальное обнаружение и История (LAN Base & Local DB)
Цель: Сделать так, чтобы две копии программы на разных ПК в одной сети видели друг друга без интернета, обменивались текстом и сохраняли историю переписки локально.

Шаг 1.1: Создание репозитория и структуры

Создай пустой проект в Visual Studio. Раздели его на три части: Lantern.Client (WPF/WinUI), Lantern.Server (Консоль) и Lantern.Shared (Общие классы).

Залей первый коммит с файлом README.md.

Шаг 1.2: Создание «Цифрового паспорта» (Идентификация)

Напиши генератор ключей (на базе System.Security.Cryptography). При первом запуске клиент создает пару ключей. Публичный ключ — это ID пользователя, приватный — хранится локально в файле user.key.

Важно: Полный сетевой адрес пользователя для работы через интернет теперь имеет вид: ID_пользователя@IP_его_сервера (по аналогии с Email).

Шаг 1.3: UDP Broadcast («Крик» в локальную сеть)

Напиши фоновый поток на C# (UdpClient), который раз в 3–5 секунд отправляет в локальную сеть пакет: «Привет, я [Мой Публичный ID], мой локальный IP [192.168.X.X]».

Настрой приемник: программа должна слушать этот порт и автоматически добавлять найденных в сети друзей в список контактов в интерфейсе.

Шаг 1.4: Первые текстовые сообщения в LAN

Сделай так, чтобы при клике на локального друга открывалось окно чата, и текстовые JSON-пакеты отправлялись напрямую на его IP-адрес через TCP/UDP.

Шаг 1.5: Локальная база данных истории чатов

Интегрируй в Lantern.Client библиотеку Microsoft.Data.Sqlite.

Создай локальный файл history.db на устройстве пользователя. При отправке или получении сообщения клиент сохраняет его в базу в расшифрованном виде, чтобы история всегда была доступна локально без участия серверов.

🟡 Фаза 2: Свой сервер и Федерация (Self-Hosted Hub & S2S)
Цель: Научить программу связываться через интернет, обеспечив надежную доставку оффлайн-сообщений через временный буфер, даже если у друзей разные VPS-серверы.

Шаг 2.1: Базовый Сигнальный сервер

Напиши легкое консольное приложение на C#. Его задача — принимать подключения от клиентов через интернет и записывать в оперативную память: «Пользователь с ID ХХХ сейчас находится на публичном IP YYY».

Шаг 2.2: Временный буфер оффлайн-сообщений (Серверная БД)

Подключи к серверу SQLite. Если ты пишешь другу, который находится в сети твоего же сервера, но он оффлайн — сервер сохраняет зашифрованную «кашу» в таблицу PendingMessages.

При включении ПК друга сервер выплескивает ему накопленные пакеты, и как только клиент подтверждает успешное получение, сервер навсегда удаляет эти сообщения со своего диска.

Шаг 2.3: Федерация серверов (Server-to-Server протокол)

Научи сервер обрабатывать внешние адреса. Если ты отправляешь оффлайн-сообщение на адрес ID_друга@IP_другого_сервера, твой VPS должен связаться через интернет с указанным IP-адресом чужого сервера и передать пакет в его базу данных.

Шаг 2.4: Упаковка в Docker и скрипт установки

Напиши Dockerfile для сервера.

Напиши простой bash-скрипт (install.sh), чтобы любой пользователь мог зайти на свой VPS, скопировать одну строчку и развернуть серверную часть Lantern за 10 секунд.

🟠 Фаза 3: Защита и Управление (Security & Admin Core)
Цель: Закрыть сервер от посторонних, защитить переписку от админа VPS с помощью E2EE и дать инструменты управления.

Шаг 3.1: End-to-End Шифрование (E2EE)

Реализуй алгоритм Diffie-Hellman (ECDH). Перед началом чата клиенты пересылают друг другу публичные ключи через серверы.

Внедри AES-256: текст сообщения шифруется на твоем ПК, серверы (даже чужие при федерации) получают «кашу» и пересылают её дальше. Владелец VPS видит в базе только зашифрованные символы.

Шаг 3.2: Токены доступа (Инвайты)

Сделай так, чтобы при запуске сервера генерировался AccessKey. Владелец сервера передает его друзьям. Без этого ключа чужой человек не сможет авторизоваться на вашем VPS.

Шаг 3.3: Админка внутри клиента

Добавь в клиентское приложение вкладку для владельца сервера. Через нее он может отправлять команды на свой VPS: установить лимит (например, максимум 5 человек), забанить кого-то по ID или посмотреть нагрузку на сеть.

🔵 Фаза 4: Встроенный VPN и Защита Сети (Overlay Network & Security)
Цель: Объединить друзей в изолированную виртуальную локальную сеть для совместных игр (например, Minecraft) и прямого P2P связи, исключив попадание посторонних.

Шаг 4.1: Интеграция WireGuard/TUN

Подключи к C#-клиенту open-source библиотеку WireGuard (или обертку над TUN/TAP драйвером).

Настрой автоматическое создание виртуального сетевого адаптера при запуске режима "Virtual LAN".

Шаг 4.2: Авто-маршрутизация пакетов

Выдели подсеть (например, 10.0.0.X). Программа должна автоматически присваивать тебе IP 10.0.0.1, другу — 10.0.0.2 и направлять весь игровой трафик через созданный зашифрованный туннель.

Шаг 4.3: Приватные комнаты VPN (Network Access Key)

Реализуй концепцию закрытых комнат. Чтобы подключиться к виртуальной LAN-сети друга, пользователь должен ввести сгенерированный владельцем ключ сети (NetworkKey). Это полностью изолирует вашу игровую сессию от других пользователей Lantern.

Шаг 4.4: Криптографический Белый Список (WhiteList)

Настрой программу так, чтобы на уровне мессенджера она автоматически игнорировала любые пакеты от устройств в виртуальной сети, чьих Публичных ID нет в твоей локальной базе друзей. Чужие люди в сети тебя не увидят и не смогут прислать спам.

Шаг 4.5: Переключатель «VPN Mode» в интерфейсе

Выведи красивый переключатель в GUI, отображающий статус туннеля, имя комнаты и список доверенных участников, которые сейчас находятся с тобой в одной «виртуальной LAN».

🟣 Фаза 5: Голос и Медиа (Hi-Fi Voice)
Цель: Добавить кристально чистый звук без задержек.

Шаг 5.1: Захват звука с микрофона

Подключи библиотеку NAudio для работы со звуковыми устройствами Windows.

Шаг 5.2: Сжатие через Opus

Интегрируй кодек Opus (через библиотеку Concentus). Голос должен сжиматься «на лету», чтобы не нагружать сеть, но оставаться качественным.

Шаг 5.3: Передача аудиопотока

Направь аудиопакеты через UDP-соединение (напрямую другу по P2P или через твой VPS, если роутеры слишком жесткие).
